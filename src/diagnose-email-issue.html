🎯 What's Happening

⚠️ Your Issue: Resend Sandbox Mode

Why only ONE email works:

  • Sandbox Mode: Using onboarding@resend.dev restricts sending
  • Verified Emails Only: Resend only sends to emails you've verified
  • Security Feature: Prevents spam during development
  • Production Block: Can't send to real users until domain verified

✅ Solutions (Choose One)

🟢 Option 1: Quick Test (2 minutes)

Best for: Testing with specific emails RIGHT NOW

Steps:

  1. Go to Resend Audiences
  2. Click "Create Audience" → "Add Contact"
  3. Add email addresses you want to test
  4. Users verify their emails
  5. ✅ Now emails will work for them!

Limitation: Only works for verified emails

🟡 Option 2: Verify Domain (30 minutes) - RECOMMENDED

Best for: Production app, unlimited emails

Steps:

  1. Go to Resend Domains
  2. Click "Add Domain" → Enter your domain
  3. Add DNS records (TXT, CNAME) to your domain provider
  4. Wait 5-10 minutes → Click "Verify"
  5. Update code: Change from: 'onboarding@resend.dev' to from: 'noreply@yourdomain.com'
  6. Deploy: supabase functions deploy make-server-40d4d8fd
  7. ✅ Send to ANYONE!

See detailed guide: HOW_TO_FIX_EMAIL_SANDBOX.md

🔵 Option 3: Use Gmail SMTP (15 minutes)

Best for: Don't have domain, need production fix

Details: See RESEND_API_SETUP.md for Gmail SMTP setup

📝 Code Changes Required (Option 2)

After verifying your domain, update these lines:

// File: /supabase/functions/server/email.tsx
// ❌ CURRENT (Line 18, 78, 142):
from: 'onboarding@resend.dev',
// ✅ CHANGE TO:
from: 'Resilio <noreply@yourdomain.com>',
// Replace 'yourdomain.com' with your actual verified domain

const senderEmail = Deno.env.get('SENDER_EMAIL') || 'noreply@yourdomain.com';
